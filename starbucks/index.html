Starbucks Redesign

A personal project I completed because I wanted a smoother and more efficient experience ordering coffee (aka magic elixir)

Hello bottom nav bar

I completely redesigned the user interface with a bottom nav bar that is found in most current mobile apps. This is to prevent user friction between using different apps and allow for a more familiar experience.

The current design of the Starbucks app is on the left and my redesign is on the right.

Quick and easy navigating

The bottom navigation provides quick, direct access to the most used functions of the application.

Problems: redundancy and unclear

A bump I found in the user experience is the store selection. The store selection option is located at the bottom of the screen throughout the entire ordering process.

While this is convenient, if the user doesn't interact with the store locator at all (in the case that the store location automatically listed from location tracking is where they want to go) the user is actually redirected to a different store locator screen entirely and prompted again to select a location, resulting in redundancy.

Another problem I had when I was ordering is the lack of knowing where I was through the entire ordering process. Is this button going to accidentally purchase my drink? Is this option going to favorite my drink? How much more before I can checkout? I wasn't sure.

My Solutions: Wizards

My solution to these problems was to have a wizard-like visual representation of the entire mobile ordering process.

I had the location selection prompted at the beginning because it would be frustrating to have to create an entire order only to find out it wasn't in stock. With location as the first option, you know right away. Each part of the selection process is clearly stated with the circular timeline at the top of the screen, users will be able to know where they are in their order and have quick access to each section if they want to make edits.

Easier transitions between selections

I also changed the UI for customizing drinks from separate screens to modals. They are also used for confirmation after a user action which helps provide confidence in selections.
